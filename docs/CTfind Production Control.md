# CTfind Production Control

Документ сконвертирован из PDF «CTfind Production Control».

## 1. Общая концепция продукта

**Название:** CTfind Production Control

**Назначение:** Система управления заказами и производством для производственных предприятий с возможностью масштабирования в платформу.

## 2. Границы MVP

**Важно:** MVP = только то, что влияет на продажи.

**Входит:**

- заказы
- производство (задачи)
- базовый склад
- роли и доступы

**Не входит:**

- финансы
- сложный MRP
- бухгалтерия
- сложные интеграции

## 3. Роли пользователей

- Администратор
- Менеджер заказов
- Мастер / начальник цеха
- Сотрудник (исполнитель)
- Склад

## 4. Функциональные требования (ядро)

### 4.1 Управление заказами

**Функции:**

- создание заказа
- редактирование заказа
- статусы:
  - новый
  - в работе
  - готов
  - отгружен
- привязка:
  - клиента
  - сроков
  - состава заказа

**Требования:**

- возможность быстрого создания (< 1 мин)
- история изменений заказа
- поиск и фильтрация

### 4.2 Управление производством (ключевой модуль)

**Функции:**

- разбиение заказа на задачи
- назначение задач сотрудникам
- статусы задач:
  - не начато
  - в работе
  - выполнено
- контроль сроков

**Требования:**

- визуализация (доска / список)
- отображение загрузки
- уведомления о просрочке

### 4.3 Управление исполнителями

**Функции:**

- список сотрудников
- назначение ролей
- привязка задач

**Требования:**

- каждый сотрудник видит только свои задачи
- учёт активности (начал/завершил)

### 4.4 Склад (упрощённый)

**Функции:**

- учёт материалов
- списание под заказ
- остатки

**Требования:**

- простой интерфейс (без сложного учёта)
- связь с заказами

### 4.5 Уведомления

**Функции:**

- просрочка задач
- изменение статуса
- новые назначения

**Каналы:**

- внутри системы (MVP)
- (опционально) Telegram / email

## 5. Базовый IAM

*Это дифференциация продукта.*

**Функции:**

1. **Ролевая модель (RBAC)**
  - разграничение доступа:
    - заказы
    - задачи
    - склад
2. **Контроль действий**
  - логирование:
    - кто создал заказ
    - кто изменил
    - кто закрыл
3. **Аудит**
  - журнал действий
  - фильтрация по пользователям

## 6. AI-функции (MVP-lite или Phase 2)

**Минимальный набор:**

1. Предупреждения
  - «заказ будет просрочен»
2. Подсказки
  - «задача не выполняется X часов»

**Расширение:**

- AI-ассистент (чат)
- прогноз загрузки

## 7. Интеграции (минимум)

**MVP:**

- API (REST)

**Phase 2:**

- 1С
- CRM

## 8. UI/UX требования

**Принципы:**

- максимально простой интерфейс
- 80% действий в 2–3 клика
- адаптация под планшеты (цех!)

**Обязательные экраны:**

- список заказов
- карточка заказа
- доска задач
- список задач сотрудника
- склад
- журнал действий

## 9. Нефункциональные требования

**Производительность:**

- до 50 пользователей (MVP)

**Доступность:**

- web (обязательно)

**Безопасность:**

- авторизация
- логирование

**Масштабируемость:**

- модульная архитектура
- API-first
