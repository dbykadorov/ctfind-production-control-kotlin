# Contract: Orders API

All endpoints are JSON-only and protected by the existing Bearer JWT authentication unless stated otherwise. Protected unauthenticated requests return `401 Unauthorized`; authenticated users without write permission return `403 Forbidden`.

## Common Types

### OrderStatus

```json
"NEW" | "IN_WORK" | "READY" | "SHIPPED"
```

Display labels:

- `NEW`: `новый`
- `IN_WORK`: `в работе`
- `READY`: `готов`
- `SHIPPED`: `отгружен`

### ApiError

```json
{
  "code": "validation_failed",
  "message": "Human readable error",
  "details": {
    "field": "reason"
  }
}
```

Common codes: `unauthorized`, `forbidden`, `not_found`, `validation_failed`, `invalid_status_transition`, `stale_order_version`.

## GET /api/customers

Search existing customers for selection.

### Query Parameters

- `search?: string`
- `activeOnly?: boolean` - defaults to `true`
- `limit?: number` - defaults to `20`, max `50`

### Response 200

```json
{
  "items": [
    {
      "id": "9ef4f2de-0c94-42d8-b968-d55d6d5b99b2",
      "displayName": "ООО Ромашка",
      "status": "ACTIVE",
      "contactPerson": "Иван Иванов",
      "phone": "+7 999 000-00-00",
      "email": "order@example.test"
    }
  ]
}
```

## GET /api/orders

List, search, and filter orders.

### Query Parameters

- `search?: string` - order number or customer text
- `status?: OrderStatus`
- `customerId?: UUID`
- `activeOnly?: boolean`
- `overdueOnly?: boolean`
- `deliveryDateFrom?: yyyy-mm-dd`
- `deliveryDateTo?: yyyy-mm-dd`
- `page?: number` - zero-based, defaults to `0`
- `size?: number` - defaults to `20`, max `100`
- `sort?: string` - defaults to a stable business order such as `updatedAt,desc`

### Response 200

```json
{
  "items": [
    {
      "id": "95849543-a34d-4d8b-828a-981f46bfb63f",
      "orderNumber": "ORD-000001",
      "customer": {
        "id": "9ef4f2de-0c94-42d8-b968-d55d6d5b99b2",
        "displayName": "ООО Ромашка",
        "status": "ACTIVE"
      },
      "deliveryDate": "2026-05-15",
      "status": "NEW",
      "statusLabel": "новый",
      "updatedAt": "2026-04-26T18:00:00Z",
      "version": 0,
      "overdue": false
    }
  ],
  "page": 0,
  "size": 20,
  "totalItems": 1,
  "totalPages": 1
}
```

## POST /api/orders

Create an order. Requires `ADMIN` or `ORDER_MANAGER`.

### Request

```json
{
  "customerId": "9ef4f2de-0c94-42d8-b968-d55d6d5b99b2",
  "deliveryDate": "2026-05-15",
  "notes": "Согласовать упаковку",
  "items": [
    {
      "itemName": "Столешница",
      "quantity": 2,
      "uom": "шт"
    }
  ]
}
```

### Response 201

Returns the full order detail. Initial status is `NEW`; `orderNumber` is generated by the backend.

## PUT /api/orders/{id}

Edit regular order fields and replace the item list. Requires `ADMIN` or `ORDER_MANAGER`. Regular edits are rejected after `SHIPPED`.

### Request

```json
{
  "expectedVersion": 0,
  "customerId": "9ef4f2de-0c94-42d8-b968-d55d6d5b99b2",
  "deliveryDate": "2026-05-20",
  "notes": "Обновленный комментарий",
  "items": [
    {
      "itemName": "Столешница",
      "quantity": 3,
      "uom": "шт"
    }
  ]
}
```

### Response 200

Returns the full updated order detail with incremented `version`.

### Error 409

```json
{
  "code": "stale_order_version",
  "message": "Order has changed. Reload before saving."
}
```

## POST /api/orders/{id}/status

Move an order through one direct forward transition. Requires `ADMIN` or `ORDER_MANAGER`.

### Request

```json
{
  "expectedVersion": 1,
  "toStatus": "IN_WORK",
  "note": "Передано в производство"
}
```

### Response 200

Returns full updated order detail with status history.

### Error 422

```json
{
  "code": "invalid_status_transition",
  "message": "Only direct forward status transitions are allowed."
}
```

## Audit Requirements

Successful `POST /api/orders`, `PUT /api/orders/{id}`, and `POST /api/orders/{id}/status` must persist:

- A business audit event with actor, target order, timestamp, and summary.
- A history/diff entry suitable for the order detail timeline.
- A status change row when status changes.

Failed validation, forbidden writes, and stale writes must not mutate order data.
